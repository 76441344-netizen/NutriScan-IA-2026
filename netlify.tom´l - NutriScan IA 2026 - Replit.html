    ✅ Correcciones aplicadas
    1. 🍽️ Generar Receta — ya no da error

    Causa raíz: El modelo de IA devolvía saltos de línea literales dentro de los strings JSON (ej. "ingredientes": "- espinaca\n- zanahoria" con un salto real, no \\n), lo que rompía JSON.parse.

    Solución aplicada:

    • Función sanitizeJson() que convierte caracteres de control literales en sus equivalentes escapados antes de parsear
    • Doble intento con temperatura más baja si el primer intento falla
    • Prompt mejorado que explícita instruye al modelo a usar \n en vez de saltos reales
    • El sistema ahora es robusto — si el modelo se equivoca, lo repara antes de fallar
    2. 📷 NutriFoto — funciona con todas las imágenes

    Causa raíz: Express tenía un límite de 100KB por request. Las imágenes base64 pesan 1–5MB → error HTTP 413.

    Solución: Límite subido a 50MB en express.json() y express.urlencoded(). También se aplicó la misma sanitización JSON a los endpoints de scans.

    3. 📅 Planner — IA sanitizada también

    Se aplicó la misma corrección de parseo JSON al endpoint de generación del planner para evitar el mismo tipo de error en esa pantalla.

    4. 🎮 NutriJuego — ahora avanza de nivel correctamente

    Causa raíz: Bug clásico de React — cierre obsoleto (stale closure). finishRound leía el estado clasificados del momento en que fue creada la función, no del momento en que fue llamada. Como resultado, siempre veía 0 respuestas correctas → nunca pasabas.

    Solución: Se agregaron refs (clasificadosRef, rachaRef, alimentosRef, phaseRef) que siempre tienen el valor más reciente, y que finishRound lee directamente. También se eliminó el doble-disparo del timer.
